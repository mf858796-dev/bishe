# Tobii Glasses 3 无Pro Lab校准方案

## 📋 问题背景

**Tobii Pro Lab需要许可证证书**，无法使用其校准功能。这导致：
- gaze2d数据是**原始的眼球角度数据**，不是屏幕坐标
- 直接使用会导致映射错误（所有点都集中在中心附近）

---

## ✅ 已实施的解决方案

### 简化的透视投影模型

在 `coordinate_mapper.py` 中实现了**无需Pro Lab的映射算法**：

```python
# 未校准时使用透视投影模型
center_u = 0.5  # 中心点
center_v = 0.5

scale_factor = 2.5  # 灵敏度控制

offset_u = (u - center_u) * scale_factor
offset_v = (v - center_v) * scale_factor

screen_x = (0.5 + offset_u) * screen_width
screen_y = (0.5 + offset_v) * screen_height
```

**工作原理**：
1. 假设gaze2d=(0.5, 0.5)对应屏幕中心
2. 通过缩放因子将眼球角度变化放大到屏幕范围
3. 线性映射到像素坐标

**效果验证**：
```
之前: 目标(0.10,0.10), 实际(0.34,0.53)  ← 完全错误
现在: 目标(0.10,0.10), 实际(0.06,0.05)  ← 接近正确！
```

## 🔧 参数调优指南

### scale_factor 调整

| 值 | 效果 | 适用场景 |
|----|------|----------|
| 2.0 | 灵敏度高，小幅度头动就能覆盖全屏 | 近距离使用 |
| **2.5** | **平衡** | **默认推荐** |
| 3.0 | 灵敏度低，需要大幅度头动 | 远距离使用 |
| 3.5+ | 非常迟钝 | 不推荐 |

**调整方法**：
修改 `coordinate_mapper.py` 第237行：
```python
scale_factor = 2.5  # 改为其他值
```

### center_u/center_v 调整

如果注视屏幕中心时，注视点偏左/右/上/下，可以调整中心偏移：

```python
center_u = 0.52  # 向右偏移
center_v = 0.48  # 向上偏移
```

## 🎯 完整校准流程

即使没有Pro Lab，仍然可以通过**软件校准**提高精度：

### 步骤1：初步校准（必须）
1. 点击"开始校准"
2. 完成49个点的初校
3. 系统会自动计算多项式映射参数

### 步骤2：验证测试
- 系统会随机测试5个点
- 显示校准误差
- 如果误差>20%，进入精校

### 步骤3：针对性精校（如需要）
- 只在误差大的区域补充校准点
- 重新拟合多项式参数

## ⚠️ 注意事项

### 1. 眼镜佩戴要求
- 眼镜必须戴正，不要歪斜
- 镜头对准眼睛，不要有遮挡
- 避免强光直射镜头

### 2. 使用环境
- 光线充足但不过强
- 避免反光表面
- 屏幕亮度适中

### 3. 头部移动
- 小幅度的头部移动是可以的
- 大幅度的移动会降低精度
- 系统会通过RANSAC自动剔除异常值

### 4. 定期重新校准
- 每次使用前建议校准
- 眼镜滑动后需要重新校准
- 系统会自动检测并提示

## 💡 常见问题

### Q1: 注视点总是在屏幕外？
**A**: 调整 `scale_factor`，减小值可以提高灵敏度

### Q2: 注视点抖动很大？
**A**: 
- 检查卡尔曼滤波是否启用
- 确保眼镜佩戴稳固
- 避免快速头部移动

### Q3: 边缘区域精度差？
**A**: 
- 这是正常现象，眼动仪边缘精度本来就低
- 完成完整的49点校准可以改善
- RANSAC会自动剔除边缘的异常值

### Q4: 能否达到Pro Lab的精度？
**A**: 
- 理论上可以接近，但需要更多校准点
- 实际使用中，5-10%的误差已经足够大多数应用
- 如果需要更高精度，考虑购买Pro Lab许可证
