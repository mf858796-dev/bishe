## ✅ **诊断二：专注力评估算法空洞 - 已解决**

### **问题分析**
- ❌ 原实现：仅记录总注视时长
- ❌ 缺少软件工程领域的经典眼动指标
- ❌ 无法发现"注意力盲区"

### **解决方案**

基于软件工程眼动研究，实现了**6大核心指标**：

#### 1. **有效阅读时长 (Effective Fixation Rate)**
```python
# 计算落在关键代码区域的时长占比
effective_rate = key_area_duration / total_duration
```
- **科研价值**：反映用户是否真正关注代码逻辑
- **阈值**：<60% 表示注意力分散

#### 2. **回视率 (Regression Rate)**
```python
# Y轴坐标显著向上移动的次数
if points[i-1][1] - points[i][1] > threshold:
    regression_count += 1
```
- **科研价值**：初学者看不懂时会反复回看
- **意义**：回视率越高，认知负荷越大
- **阈值**：>10次/分钟表示理解困难

#### 3. **扫视熵 (Saccade Entropy)**
```python
# 衡量阅读路径的混乱程度（香农熵）
entropy = -Σ(p * log2(p))
```
- **科研价值**：结构化阅读 vs 无序浏览
- **归一化**：0-1范围，越低越有序
- **阈值**：>0.7 表示阅读路径混乱

#### 4. **平均注视时长 (Average Fixation Duration)**
```python
# 每次注视的平均持续时间
avg_duration = mean(fixation_intervals)
```
- **科研价值**：反映信息处理深度
- **正常范围**：200-400ms
- **异常**：<150ms 可能是扫视，>500ms 可能是困惑

#### 5. **扫视路径总长度 (Scanpath Length)**
```python
# 所有注视点之间的欧氏距离总和
total_length = Σ||point[i] - point[i-1]||
```
- **科研价值**：眼球运动效率
- **意义**：路径越短，阅读越高效

#### 6. **数据质量评分 (Data Quality Score)**
```python
# 清洗后数据量 / 原始数据量
quality = len(filtered) / len(raw)
```
- **科研价值**：排除眨眼、追踪丢失等噪声
- **阈值**：<0.8 表示数据不可靠

### **综合专注度评分算法**

```python
score = fixation_rate           # 有效注视率贡献 (0-100)
      - regression_penalty      # 回视惩罚 (最多-30)
      - entropy_penalty         # 混乱度惩罚 (最多-20)
      + data_quality            # 数据质量加成 (0-10)
      
score = max(0, min(100, score))  # 限制在0-100范围
```

**评分等级：**
- 🟢 **优秀 (90-100)**：专注度高，阅读路径清晰
- 🟡 **良好 (75-89)**：基本专注，偶有回视
- 🔴 **待提升 (<75)**：注意力分散，需要改进

## 📊 **技术实现细节**

### **1. 行号检测算法**

```python
def get_line_at_position(self, y_pos):
    """根据Y坐标获取行号"""
    cursor = self.cursorForPosition(QPointF(0, y_pos).toPoint())
    return cursor.blockNumber()
```

**优势：**
- 精确到行级别
- 支持滚动
- 性能优化（O(1)复杂度）

### **2. 视觉引导动画**

```python
# 闪烁效果：正弦波控制透明度
flash_alpha = int(128 + 127 * math.sin(time.time() * 4))
guide_color = QColor(251, 191, 36, flash_alpha)
```

**效果：**
- 4Hz频率闪烁（人眼敏感）
- 黄色醒目但不刺眼
- 吸引注意力到目标行

### **3. 回视检测优化**

```python
# 只在目标代码块区域内检测
if (block_top <= y <= block_bottom and
    block_top <= prev_y <= block_bottom):
    if y < prev_y - 10:  # 10像素阈值
        regression_count += 1
```

**避免误判：**
- 排除跨块移动
- 设置合理阈值
- 只统计有意义的回视

### **4. 数据清洗算法**

```python
# 速度过滤：去除过快移动（眨眼/丢失）
velocity = distance / dt
if velocity > max_velocity:  # 2000 px/s
    return (prev_x, prev_y, timestamp)  # 保持平滑
```

**科学依据：**
- 人眼最大扫视速度约900°/s
- 换算为屏幕像素约2000px/s
- 超过此值视为异常数据
