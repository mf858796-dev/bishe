# 训练模块实时注视点显示功能修复说明

## 问题描述
训练模块已添加实时注视点显示功能，但开始训练后没有显示注视点。

## 问题原因分析

1. **坐标转换错误**：原来的代码使用 `mapFromParent` 方法转换坐标，但这个方法不适用于屏幕坐标到组件坐标的转换。

2. **重复调用 check_gaze**：`main_window.py` 中的 `update_system_status` 和 `on_gaze_mapped` 两个方法都在调用 `check_gaze`，导致逻辑混乱。

3. **连接状态限制**：只有在眼镜连接状态下才会更新训练模块的注视点，限制了测试和使用。

4. **组件大小未正确设置**：注视点组件在初始化时设置了大小，但当窗口调整大小时没有同步更新。

## 修复内容

### 1. training_widget.py 修改

#### 修改坐标转换逻辑（第607-630行）
```python
# 修复前：
editor_pos = self.code_editor.mapFromParent(QPointF(x, y))
self.gaze_point_widget.add_gaze_point(editor_pos.x(), editor_pos.y())

# 修复后：
# 获取代码编辑器相对于主窗口的位置
editor_global_pos = self.code_editor.mapToGlobal(self.code_editor.rect().topLeft())
# 将屏幕坐标转换为相对于代码编辑器的坐标
editor_x = x - editor_global_pos.x()
editor_y = y - editor_global_pos.y()

# 检查是否在代码编辑器范围内
if (0 <= editor_x <= self.code_editor.width() and 
    0 <= editor_y <= self.code_editor.height()):
    print(f"[DEBUG] 添加注视点: 屏幕({x:.0f},{y:.0f}) -> 编辑器({editor_x:.0f},{editor_y:.0f})")
    self.gaze_point_widget.add_gaze_point(editor_x, editor_y)
else:
    print(f"[DEBUG] 注视点超出编辑器范围: 编辑器({editor_x:.0f},{editor_y:.0f}), 大小({self.code_editor.width()}x{self.code_editor.height()})")
```

#### 修改组件初始化（第217-220行）
```python
# 修复前：
self.gaze_point_widget = GazePointWidget(self.code_editor)
self.gaze_point_widget.setGeometry(self.code_editor.rect())
self.gaze_point_widget.hide()
self.code_editor.resizeEvent = self.on_code_editor_resize

# 修复后：
self.gaze_point_widget = GazePointWidget(self.code_editor)
self.gaze_point_widget.hide()  # 初始隐藏
# 注意：大小将在 showEvent 中设置
```

#### 添加 showEvent 方法（第232-237行）
```python
def showEvent(self, event):
    """窗口显示时设置注视点组件大小"""
    super().showEvent(event)
    if self.gaze_point_widget:
        # 确保注视点组件与代码编辑器大小一致
        self.gaze_point_widget.setGeometry(self.code_editor.rect())
```

#### 增强 start_training 方法（第504-523行）
```python
# 显示注视点组件并确保大小正确
if self.gaze_point_widget:
    # 强制更新几何形状
    self.gaze_point_widget.setGeometry(self.code_editor.rect())
    self.gaze_point_widget.show()
    self.gaze_point_widget.clear_points()
    print(f"[DEBUG] 注视点组件已显示, 大小: {self.gaze_point_widget.size()}")
    print(f"[DEBUG] 代码编辑器大小: {self.code_editor.size()}")
```

### 2. main_window.py 修改

#### 移除 update_system_status 中的重复调用（第656-658行）
```python
# 修复前：
def update_system_status(self):
    """更新系统状态"""
    self.update_metrics_display()
    
    # 只有在已连接且正在训练时才调用 check_gaze
    if hasattr(self, 'current_gaze') and self.is_connected:
        x, y = self.current_gaze
        now = time.time()
        dt = now - self.last_time
        self.last_time = now
        self.training_widget.check_gaze(x, y, dt)

# 修复后：
def update_system_status(self):
    """更新系统状态"""
    self.update_metrics_display()
```

#### 修改 on_gaze_mapped 方法（第811-815行）
```python
# 修复前：
# 更新训练模块的 gaze 检测
if hasattr(self, 'training_widget') and self.is_connected:
    dt = time.time() - self.last_time
    self.last_time = time.time()
    self.training_widget.check_gaze(x, y, dt)

# 修复后：
# 更新训练模块的 gaze 检测（无论是否连接眼镜，只要有gaze数据就更新）
if hasattr(self, 'training_widget'):
    dt = time.time() - self.last_time
    self.last_time = time.time()
    self.training_widget.check_gaze(x, y, dt)
```

## 使用方法

### 方法1：启动完整应用
```bash
python main.py
```

启动后：
1. 切换到"🎯 训练任务"标签页
2. 选择一个训练任务
3. 点击"▶️ 开始训练"按钮
4. 当有注视点数据时，会在代码编辑器上显示红色圆点

### 方法2：单独测试训练模块
```bash
python test_training_gaze.py
```

这个测试程序会：
- 只显示训练模块
- 提供一个测试按钮来模拟注视点
- 在控制台输出调试信息

## 注意事项

1. **注视点坐标系统**：
   - 输入的是屏幕绝对坐标（相对于整个屏幕）
   - 会自动转换为代码编辑器内的相对坐标
   - 只有当注视点在代码编辑器范围内时才会显示

2. **显示效果**：
   - 最新的注视点显示为较大的红色圆圈
   - 旧的注视点会逐渐变小并变透明
   - 最多显示20个点，每个点存活2秒

3. **性能优化**：
   - 移除了重复的 `check_gaze` 调用
   - 只在 `on_gaze_mapped` 中处理注视点数据
   - 避免了不必要的计算

## 常见问题

### Q1: 开始训练后仍然看不到注视点？
**A**: 请检查：
1. 控制台是否有调试信息输出
2. 注视点坐标是否在代码编辑器范围内
3. 是否真的有注视点数据传入（需要连接眼镜或使用测试程序）

### Q2: 注视点位置不正确？
**A**: 这可能是坐标转换的问题。检查调试信息中的坐标值，确保：
- 屏幕坐标是正确的（通常是 0-1920 x 0-1080）
- 编辑器坐标在合理范围内（0 到编辑器宽度/高度）

### Q3: 如何验证功能是否正常？
**A**: 运行测试程序 `test_training_gaze.py`，点击测试按钮，应该能看到：
- 控制台输出调试信息
- 代码编辑器上出现红色圆点

## 下一步改进建议

1. **添加鼠标追踪测试**：可以添加鼠标移动事件监听，用鼠标位置模拟注视点，方便测试。

2. **优化坐标转换**：如果窗口可以移动或调整大小，需要实时更新代码编辑器的全局位置。

3. **添加视觉反馈**：可以在注视点进入目标区域时改变颜色（如变为绿色），提供更直观的反馈。

4. **性能监控**：添加帧率显示，确保注视点渲染不会影响整体性能。
