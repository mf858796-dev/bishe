# 眼动仪集成说明

## 📋 概述

训练模块现在已经完全支持 Tobii Pro Glasses 3 眼动仪。当你戴上眼动仪并注视代码时，系统会：
1. 接收眼动仪的屏幕坐标
2. 自动转换为代码编辑器的内部坐标
3. 在代码上显示红色注视点
4. 统计有效注视时间和完成率

### 1. 坐标转换系统
- ✅ 添加了 `convert_screen_to_viewport()` 方法
- ✅ 将屏幕绝对坐标转换为代码编辑器 viewport 坐标
- ✅ 自动处理窗口位置和偏移量

### 2. 训练逻辑优化
- ✅ 简化了 `check_gaze()` 方法
- ✅ 统一使用 viewport 坐标系统
- ✅ 移除了复杂的坐标转换逻辑

### 3. 测试工具
- ✅ `test_mouse_gaze.py` - 鼠标模拟测试（无眼动仪时使用）
- ✅ `test_gaze_integration.py` - 眼动仪集成测试

1. **连接眼动仪**
   ```bash
   # 确保 Tobii Pro Glasses 3 已连接到电脑
   # USB 连接或 WiFi 连接均可
   ```


   - 控制台会显示：`[眼动] 屏幕坐标: (xxx, yyy)` 和 `[眼动] Viewport坐标: (xxx, yyy)`

   - 右侧统计面板会显示有效注视时间增长
   - 完成率会逐步提升

### 方式二：使用鼠标模拟（无眼动仪时）

   - 观察红色注视点跟随鼠标移动

## 📊 坐标系统说明

### 三种坐标系统

1. **屏幕坐标**（眼动仪原始数据）
   - 范围：`(0, 0)` 到 `(屏幕宽度, 屏幕高度)`
   - 例如：`(1920, 1080)` 表示屏幕右下角

2. **TrainingWidget 坐标**
   - 相对于 TrainingWidget 窗口左上角
   - 包含标题栏、任务信息等所有组件

3. **Viewport 坐标**（最终使用）
   - 相对于代码编辑器内容区域左上角
   - 范围：`(0, 0)` 到 `(viewport宽度, viewport高度)`
   - 例如：`(475, 14)` 表示代码第一行的中心

### 转换流程

```
眼动仪 → 屏幕坐标 → TrainingWidget坐标 → Viewport坐标 → 训练逻辑
         (on_gaze_mapped)  (convert_screen_to_viewport)  (check_gaze)
```

## 🔍 调试技巧

### 查看坐标转换是否正常

控制台会输出：
```
[眼动] 屏幕坐标: (960, 540)
[眼动] Viewport坐标: (475, 280)
```

如果只看到屏幕坐标，没有 viewport 坐标，说明：
- 训练未开始（需要先点击"开始训练"）
- 注视点不在代码区域内

### 检查有效注视是否工作

观察右侧统计面板：
- **总注视点**：应该持续增加
- **有效注视**：只有在注视目标代码块时才增加
- **完成率**：随着有效注视时间增长而提升

### 常见问题

**问题1：注视点不显示**
- 确认已点击"开始训练"
- 确认 gaze_point_widget 已创建
- 检查控制台是否有错误信息

**问题2：有效注视不增长**
- 确认注视的是当前高亮的代码块
- 检查坐标转换是否正确（查看控制台输出）
- 确认目标区域检测逻辑正常

**问题3：坐标对不上**
- 检查窗口是否被移动过
- 确认屏幕分辨率设置正确
- 尝试重启程序

## 📝 技术细节

### 关键文件

1. **main_window.py**
   - `on_gaze_mapped()`: 接收眼动数据
   - `convert_screen_to_viewport()`: 坐标转换

2. **training_widget.py**
   - `check_gaze()`: 处理注视点，更新训练状态
   - `GazePointWidget`: 绘制红色注视点

3. **test_mouse_gaze.py**
   - 鼠标模拟测试工具
   - 自动测试功能

### 核心算法

坐标转换公式：
```python
vp_x = screen_x - (window_x + editor_x + viewport_x)
vp_y = screen_y - (window_y + editor_y + viewport_y)
```

其中：
- `window_x, window_y`: TrainingWidget 在屏幕上的位置
- `editor_x, editor_y`: 代码编辑器在 TrainingWidget 中的位置
- `viewport_x, viewport_y`: viewport 在代码编辑器中的位置（通常为 0, 0）
