# Tobii Pro Glasses 3 USB 连接故障排查指南

## 问题描述
通过 USB 连接 Tobii Pro Glasses 3 时,软件提示"连接超时/找不到设备"。

## 已修复的问题

### 1. 代码错误修复
**问题**: `glasses_manager.py` 中使用了不存在的方法 `Glasses3.connect_to_first_found()`

**修复**: 
- 替换为正确的 `connect_to_glasses.with_zeroconf()` 和 `connect_to_glasses.with_hostname()`
- 添加了两种连接模式支持:自动发现(Zeroconf)和固定 IP

### 2. API 调用修正
**问题**: 使用了错误的 API 方法名
- `get_serial_number()` -> `get_recording_unit_serial()`
- `get_firmware_version()` -> `get_version()`

**修复**: 已更新所有相关调用

### 3. UI 增强
**新增功能**:
- 在状态栏添加连接方式选择下拉框
- 支持切换"自动发现(WiFi)"和"指定 IP(USB)"模式
- IP 地址输入框(默认显示 192.168.75.51)

## USB 连接配置步骤

### 步骤 1: 检查硬件连接

1. **确认眼镜已开机**
   - 长按电源键 2 秒
   - 等待指示灯变为绿色

2. **检查 USB 连接**
   - 使用 USB-C 数据线连接眼镜和电脑
   - 确保线缆支持数据传输(不仅仅是充电)

### 步骤 2: 安装驱动程序

Tobii Pro Glasses 3 通过 USB 连接时会创建一个虚拟网卡:

1. **Windows 系统**
   - 打开"设备管理器"
   - 展开"网络适配器"
   - 查找包含 "Tobii" 或 "Remote NDIS" 的设备
   - 如果看到黄色感叹号,需要安装驱动

2. **驱动程序位置**
   - 通常随 Tobii Pro Lab 软件自动安装
   - 或从 Tobii 官网下载: https://www.tobiipro.com/support/downloads/

### 步骤 3: 检查网络配置

USB 连接实际上是通过 USB 网络接口通信的:

1. **查看网络连接**
   ```
   控制面板 > 网络和共享中心 > 更改适配器设置
   ```
   - 应该能看到一个新的网络连接(可能是"Tobii"或"以太网"名称)

2. **检查 IP 配置**
   - 右键点击该连接 > 属性
   - 双击"Internet 协议版本 4 (TCP/IPv4)"
   - 确认设置为自动获取 IP,或者手动设置为:
     - IP 地址: 192.168.75.x (x 不等于 51)
     - 子网掩码: 255.255.255.0
     - 网关: 192.168.75.51

3. **测试连接**
   打开命令提示符,运行:
   ```bash
   ping 192.168.75.51
   ```
   如果能 ping 通,说明网络连接正常。

### 步骤 4: 配置软件

1. **启动应用程序**
   ```bash
   venv\Scripts\python.exe main.py
   ```

2. **选择连接方式**
   - 在状态栏找到"连接方式"下拉框
   - 选择"🎯 指定 IP (USB)"
   - 确认 IP 地址为 `192.168.75.51`(如果不是,请修改)

3. **点击连接按钮**
   - 点击"🔌 连接 Tobii Pro Glasses 3"
   - 观察状态栏变化

### 步骤 5: 运行测试脚本

如果仍然无法连接,运行诊断测试:

```bash
venv\Scripts\python.exe test_connection.py
```

该脚本会:
- 测试 Zeroconf 自动发现模式
- 测试固定 IP 连接模式
- 输出详细的错误信息

## 常见问题及解决方案

### 问题 1: Ping 不通 192.168.75.51

**可能原因**:
- USB 驱动未正确安装
- 防火墙阻止了连接
- 眼镜未处于可连接状态

**解决方法**:
1. 重新插拔 USB 线
2. 重启眼镜
3. 暂时关闭 Windows 防火墙测试
4. 检查设备管理器中的驱动状态

### 问题 2: 连接超时

**可能原因**:
- IP 地址不正确
- 眼镜已被其他程序占用(Tobii Pro Lab)

**解决方法**:
1. 关闭 Tobii Pro Lab 等其他连接眼镜的程序
2. 确认 IP 地址是否正确
3. 尝试重启应用

### 问题 3: Zeroconf 发现失败

**可能原因**:
- Windows 防火墙阻止了 mDNS/Bonjour 服务
- 网络配置问题

**解决方法**:
- 改用"指定 IP"模式连接
- 或在防火墙中允许 UDP 5353 端口(mDNS)

### 问题 4: 数据流错误

**成功连接但无法接收数据**:
- 确保眼镜已佩戴(某些传感器需要检测到佩戴)
- 检查电池电量是否充足
- 确认眼镜固件版本兼容性

## 连接流程图

```
开始
  |
  ├─> 眼镜已开机?
  |     └─> 否: 长按电源键 2 秒开机
  |
  ├─> USB 已连接?
  |     └─> 否: 使用 USB-C 线连接
  |
  ├─> 驱动已安装?
  |     └─> 否: 安装 Tobii USB 驱动
  |
  ├─> 能 ping 通 192.168.75.51?
  |     └─> 否: 检查网络配置
  |
  ├─> 选择"指定 IP"模式
  |
  ├─> 点击连接
  |
  └─> 成功!
```

## 技术支持

如果以上步骤都无法解决问题:

1. **收集诊断信息**
   - 运行 `test_connection.py` 的输出
   - 设备管理器中的网络适配器截图
   - `ipconfig /all` 的输出

2. **联系支持**
   - Tobii 技术支持: support@tobii.com
   - 或在项目中提交 Issue
